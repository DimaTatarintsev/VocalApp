Дизайн и интерфейс (UI/UX)



1. Figma (https://www.figma.com)

   · Для чего: Основной инструмент для создания дизайн-макетов, прототипов и UI-китов. Бесплатен для индивидуальных проектов.

   · Что искать: Готовые UI-киты для мобильных приложений, компоненты для iOS и Android.

2. Material Design (https://m3.material.io) и Apple Human Interface Guidelines (https://developer.apple.com/design/)

   · Для чего: Официальные гайдлайны от Google и Apple. Помогут сделать интерфейс привычным и удобным для пользователей каждой платформы.

3. Icons8 (https://icons8.com) и Flaticon (https://www.flaticon.com)

   · Для чего: Библиотеки иконок для любого раздела приложения (микрофон, ноты, метроном, настройки).



Разработка и документация



1. React Native (https://reactnative.dev) или Flutter (https://flutter.dev)

   · Для чего: Официальная документация по выбранному фреймворку. Начинайте всегда отсюда.

2. Stack Overflow (https://stackoverflow.com)

   · Для чего: Решение 99% возникающих технических проблем и ошибок. Велика вероятность, что с вашей проблемой уже столкнулись.

3. GitHub (https://github.com)

   · Для чего:

     · Хранение кода и контроль версий.

     · Поиск готовых библиотек и примеров кода (например, по запросам react-native audio, pitch detection).



Библиотеки и готовые решения (Критически важны)



1. npmjs.com (для React Native) / pub.dev (для Flutter)

   · Для чего: Поиск и установка библиотек для конкретных задач.

   · Примеры запросов для поиска:

     · Аудио запись: react-native-audio-record, react-native-sound, flutter_sound

     · Визуализация аудио: react-native-audio-waveform, wavesurfer.js (через WebView)

     · Визуализация графиков: react-native-svg, react-native-chart-kit, fl_chart (для Flutter)

     · Анализ высоты тона (Pitch Detection): Это сложнее. Ищите готовые решения, например, @tidal-engineering/react-native-pitch-detector или библиотеки для нативной разработки, которые можно связать с вашим проектом (Aubio - https://aubio.org).



Работа с аудио (алгоритмы и теория)



1. Aubio (https://aubio.org)

   · Для чего: Очень мощная и популярная библиотека на C для анализа звука, включая определение высоты тона (pitch), темпа (tempo) и др. Можно найти готовые биндинги (обертки) для мобильных платформ или вдохновиться алгоритмами.

2. The Audio Programmer (https://theaudioprogrammer.com) и Music DSP (https://www.musicdsp.org)

   · Для чего: Сообщества и ресурсы, посвященные цифровой обработке звука (Digital Signal Processing - DSP). Помогут глубже понять, как работают алгоритмы анализа тона.



Контент и наполнение



1. OpenAI Whisper (https://openai.com/index/whisper/)

   · Для чего: Если планируете добавлять расшифровку текста из вокала или подсказки. Это мощная модель для распознавания речи.

2. Музыкальная теория для программистов (https://www.musictheoryforprogrammers.com/)

   · Для чего: Поможет структурировать знания о нотах, интервалах и гаммах, которые нужны для создания эталонов для сравнения.



Публикация



1. Google Play Console (https://play.google.com/console) и Apple Developer Program (https://developer.apple.com)

   · Для чего: Официальные порталы для публикации приложения в магазинах. Изучите требования к скриншотам, описаниям и политикам заранее.



Вдохновение и анализ



1. AppFollow (https://appfollow.io) и Sensor Tower (https://sensortower.com)

   · Для чего: Анализ отзывов и статистики ваших конкурентов. Бесплатного функционала хватит, чтобы понять, что пользователям нравится или не нравится в аналогичных приложениях.
